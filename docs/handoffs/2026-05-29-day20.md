## Process lessons

**1. Reads first when resuming after a break.** Returning after the 4-day Eid gap, my instinct was to "continue from where the conversation ended." Instead the first action was four read-only commands (git log, git status, ls docs/handoffs, ls sql/migrations). Those surfaced three things: uncommitted soft-delete work, no Day 19 handoff, and a 4-day close backlog. Without the reads we would have written features on top of unstable ground. Lesson: any session-resume crossing a real time boundary starts with "what's the actual state right now."

**2. Claude Code auto-commits past explicit "do not commit" instructions.** Wrote "STOP. DO NOT commit." in the Day 19 handoff create-file instructions; Claude Code wrote the file, then committed and pushed it as `12cb7cb` unprompted. Going forward using explicit STEP N: numbering and stating "do not git add, git commit, or git push. I will send a separate STEP for those." Hasn't fully eliminated the behavior but reduces it.

**3. Admin-level fixes are fine when documented as one-off.** The temptation when finding a missed-state condition is to either ignore it or treat it as a feature gap and build the proper UI. Both wrong if the situation isn't likely to recur. The May 25-28 backlog was a vacation gap with zero activity — one-off SQL is the right tool. Catch-up UI stays queued for real catch-up scenarios.

**4. Audit columns are simpler than they look when the immutability trigger does the gating.** Initial worry: do audit columns get populated for failed edits? Resolution: the trigger blocks the entire UPDATE atomically. No partial writes. Audit population only happens on successful edits — by construction on live rows. No special handling needed.

**5. Cleanup before features.** Front-loading the small risks (git identity, locale format, closed-day disable) on top of the larger feature (audit trail) meant each subsequent step built on a clean state. The total cleanup took ~50 minutes; the value of a clean Open Questions list at end of shift is higher.
