## Process lessons reinforced today

1. **Production-vs-repo schema check first.** Before drafting R-A and I-B migrations: `SELECT column_name FROM information_schema.columns WHERE table_name=...`. Applied yesterday's lesson, saved time.

2. **Read existing imports/state/JSX before targeting str_replace.** Every str_replace today started with a verbatim sed/grep to confirm the anchor matches byte-for-byte. Prevented "old_str doesn't match" errors that cost ~20 min on Day 21.

3. **Split Claude Code prompts from Supabase SQL.** Mid-shift, user explicitly requested separation. Claude was mixing them. Going forward: Claude Code blocks and Supabase SQL blocks clearly labeled and separated; never combined in one prompt.

4. **Claude Code autonomy drift recurs.** Mid-session twice (inventory.md non-execution; I-D commit summary), Claude Code paraphrased a prompt instead of running it. Caught via ls/git verification both times. Pattern: when Claude Code's reply reads like a summary of intent rather than literal output, verify directly via separate read commands before proceeding.

5. **Service-layer flat selects + UI lookup maps.** Continued Day 21 pattern across R-C and all inventory work. No PostgREST nested-join typing issues.

6. **Pushback on overscoped requests.** User said "get everything done today" for I-C; Claude pushed back with honest scope analysis (5+ hours not 3) and recommended deferring issue/damage/transfer UI to Day 23. User accepted. Avoided likely Day 21-style mid-shift design churn.

7. **Constraint scope expansion requires constraint rename.** Renamed chk_account_transactions_expense_out_integrity → chk_account_transactions_revenue_expense_integrity to honestly reflect expanded scope. Small thing, big future-readability win.

8. **Mirror discipline for parallel concepts.** Revenue mirrors Expense; Inventory categories mirror both. Same shape, color theming distinguishes (amber/emerald/indigo). User can predict where things live. Reduces design surface.
